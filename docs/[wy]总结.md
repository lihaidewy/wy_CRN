  ---
  总：各优化方法详解

  CRN 框架由四大模块构成：图像编码与视图变换（RVTLSSFPN）→ 雷达点云编码（PtsBackbone）→ 多模态融合（MFAFuser）→
  检测头（BEVDepthHead）。针对路侧场景远端（100m 以上）漏检率高、纵向定位精度差的问题，480
  格配置从三个模块切入，部署了五项优化。

  ---
  优化一：BEV 纵向分辨率提升（作用于 RVTLSSFPN 视图变换 & 全网络）

  BEV 网格的 X 轴（纵向）分辨率从早期的 1.25m/格提升至 0.5m/格（480 格），Y 轴（横向）维持 0.4m/格（128
  格）。这是因为路侧场景的感知重点是纵向 0~240m 范围内的目标定位——横向偏差（±25.6m
  内）天然较小，而纵向误差随距离非线性放大。将 X 轴分辨率从 1.25m 翻 2.5 倍至 0.5m，意味着每个 0.5m 的距离区间有独立的
  BEV 特征网格，检测头在进行 heatmap 峰值定位和 BBox 回归时有更细粒度的空间分辨率可用。此项优化通过
  backbone_img_conf['x_bound'] = [0.0, 240.0, 0.5] 配置，影响 LSS 深度分布离散化、BEV 特征图尺寸（480×128）、以及
  MFAFuser 中 bev_shape 和 ref_2d 参考点的数量，是整个网络的空间精度基础。

  ---
  优化二：10 维回归 Code Weights 调优（作用于 BEVDepthHead BBox 回归损失）

  CenterPoint 风格检测头对每个目标输出 10 维回归参数：[dx, dy, z, w, l, h, sin(yaw), cos(yaw), vx, vy]。code_weights
  向量对不同维度的 SmoothL1 Loss 施加差异化权重，直接影响优化器对各维度的梯度分配。当前配置为 [4.0, 2.0, 0.5, 1.5, 1.5,
  0.5, 3.0, 3.0, 0.2, 0.2]，设计逻辑如下：

  ┌──────────────────────┬──────┬────────────────────────────────────────────────────────────────────────────────────┐
  │         维度         │ 权重 │                                        理由                                        │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────────────────────────────────────┤
  │ dx（纵向偏移）       │ 4.0  │ 核心指标，纵向误差随距离放大效应最强，用最高权重强制优化器优先减小纵向残差         │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────────────────────────────────────┤
  │ dy（横向偏移）       │ 2.0  │ 次于纵向，路侧横向 ±25.6m 内偏差天然可控，但需保持一定精度防止横向漂移             │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────────────────────────────────────┤
  │ sin/cos(yaw)（朝向） │ 3.0  │ 朝向精度直接影响 3D 框的覆盖质量——yaw 偏差 5° 在 200m 处可产生 17m                 │
  │                      │      │ 的角点偏移，高权重确保框的朝向与目标真实朝向对齐                                   │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────────────────────────────────────┤
  │ w, l（长宽尺寸）     │ 1.5  │ 路侧目标尺寸变化范围有限，适中权重                                                 │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────────────────────────────────────┤
  │ z, h（高度）         │ 0.5  │ 路侧路面平坦，目标高度变化小                                                       │
  ├──────────────────────┼──────┼────────────────────────────────────────────────────────────────────────────────────┤
  │ vx, vy（速度）       │ 0.2  │ 毫米波雷达自带径向速度测量，检测头速度回归为辅助任务                               │
  └──────────────────────┴──────┴────────────────────────────────────────────────────────────────────────────────────┘

  ---
  优化三：距离感知 BBox 回归实例加权（作用于 BEVDepthHead BBox 回归损失）

  在 bev_depth_head_det.py 的 get_targets_single() 中，为每个 GT 目标计算一个 dist_weight 标量，乘入 BBox
  回归损失的逐实例权重矩阵。距离越远的目标，回归损失被放大越多，迫使优化器在远端投入更多"注意力"。具体计算公式为：

  bbox_weights = mask × code_weights × dist_weight  （逐实例广播）
  loss_bbox = SmoothL1(pred, target, weight=bbox_weights)

  其中 dist_weight 在 Fuser 中已按距离分段预设（见分述部分），mask 过滤无效目标。此项优化的物理直觉是：同样 1m
  的纵向定位误差，在 50m 处和 200m 处对 BEV 热力图上的像素偏移量完全不同——200m 处 1m 误差仅对应 2 个 BEV
  像素（0.5m/格），而 50m 处对应 2 个像素但感知重要性和安全冗余完全不同。通过学习率的等效缩放效应，高 dist_weight
  相当于对远端样本施加更大的学习率，弥补远端样本在 batch 中天然稀疏的劣势。

  ---
  优化四：Gaussian Radius 分段扩大（作用于 BEVDepthHead 热力图目标分配）

  CenterPoint 在生成训练热力图真值时，对每个 GT 目标的中心位置画一个高斯核，高斯半径由目标尺寸和 gaussian_overlap
  参数（当前 0.1）共同决定。bev_depth_head_det.py 在第 254-272 行对这个基础 radius 做分段放大：按目标的纵向距离 x
  分四档，乘以 1.0×（<50m）到 2.0×（>150m）的系数。扩大 radius 的效果是：

  1. 热力图正样本区域扩大：远端目标的高斯核覆盖更多 BEV 像素，增加了 Focal Loss
  中正样本的占比，缓解正负样本极度不均衡（远端目标在 480×128=61440 个 BEV 网格中可能仅占 1-2 个像素）；
  2. 增大中心点定位容错：远端 depth 估计误差导致目标投影位置在 BEV
  上偏移，更大的高斯核给予检测头更大的"命中"容错区间，避免因 depth 噪声导致 GT 热力点落在目标真实 BEV
  位置的邻域之外而被当作负样本惩罚；
  3. 正向引导收敛：更大的监督区域在训练早期提供更强的梯度信号，引导检测头先"找到"远处有目标，再逐步精确其位置。

  ---
  优化五：距离感知深度损失（作用于 RVTLSSFPN DepthNet 深度预测）

  base_exp.py 中的 get_depth_loss_distance_aware() 替代了原有的均匀 BCE 深度损失。其核心机制是：对 DepthNet 输出的每个
  BEV 像素，根据其 GT 深度值所属的 bin 区间，赋予不同的 BCE Loss 权重——远端像素（bins 80-119，对应深度 162-240m）权重
  2.0×，中端（bins 40-79，82-160m）1.0×，近端（bins 0-39，2-80m）0.5×。具体计算流程为：

  gt_bins = argmax(depth_gt_soft_labels)       # 每像素的 GT 深度 bin
  pix_weights = f(gt_bins)                      # 按 bin 查表得权重
  per_pixel_bce = mean(BCE(pred, gt), dim=D)    # 120 个深度 bin 的 BCE 取均值
  loss_depth = sum(per_pixel_bce × pix_weights) / sum(pix_weights)  # 加权归一化

  此项优化针对的是 CRN 框架的视图变换模块——LSS 的深度分布预测是图像特征从 2D 提升到 3D 的关键环节。毫米波雷达在 200m
  以上返回的点云极其稀疏（可能仅 1-3 个点），导致深度 GT 在远端几乎无监督信号。通过给远端像素 2.0× 的 loss 权重，迫使
  DepthNet 在远端区域投入更多模型容量学习准确的深度分布，间接提升视图变换后 BEV
  特征中远端目标的空间位置编码质量，最终改善检测头对远端目标的定位精度。

  ---
  辅助优化：训练超参数调优（作用于训练流程）

  ┌────────────────────────┬───────────┬─────────────────────────────────────────────────────────────────────────────┐
  │          参数          │    值     │                                    作用                                     │
  ├────────────────────────┼───────────┼─────────────────────────────────────────────────────────────────────────────┤

  ---
  分：场景一（Frontal）与场景二（Oblique）的差异化策略

  场景一 Frontal（660 帧训练集，yaw=0° 正向安装，雷达高度 7m）

  Frontal
  场景的核心特征是数据量相对充足且成像几何简单（相机正对车流方向，目标在图像中分布均匀）。优化策略以保守稳健为主线：

  - BBox 回归 dist_weight
  分段：远端（>150m）2.5×、过渡段（100-150m）2.0×、中间段（50-100m）2.0×、近端（<50m）1.2×。设计特点是中间段与过渡段同为
  2.0×，而非 Oblique 的 1.5×→2.0× 递增——因为 Frontal
  场景下中间段（50-100m）的目标密度最高、承载主要的评估指标贡献，保持 2.0×
  高权重确保优化器不因远端加码而忽视中间段。远端仅用 2.5×（而非 Oblique 的
  3.0×），避免远端少数样本的梯度压倒中间段大量样本。
  - Heatmap Focal Loss 加权：关闭。在 loss() 函数中，hw = torch.ones_like(heatmap_weights)，即热力图损失权重矩阵全置
  1。原因是若开启距离加权（hw = heatmap_weights = dist_weight），远端 1 个目标的热力图中心像素获得 2.5×
  的分类损失权重，而中间段 10 个目标各获
  2.0×——虽然单目标权重更小，但总数占优，看起来不应有问题。然而实际训练中，远端目标的 heatmap 高斯核覆盖的 BEV 像素数经
  radius 扩大后显著增多，每个远端目标贡献的正样本像素数远多于近端目标，乘以 2.5× 后会产生"虹吸效应"——Focal Loss
  的梯度被远端少量目标支配，中间段目标的分类精度下降，导致整体 MOTA 反而恶化。关闭 heatmap
  加权后，分类监督信号保持空间均匀，配合 BBox 回归的 dist_weight 在回归分支独立处理距离差异，实现分与回归的解耦优化。
  - Gaussian Radius：保守的 2.0×/1.5×/1.2×/1.0×（与 Oblique
  相同系数），不做激进扩大以免过度的正样本扩散稀释近端目标的定位精度。

  场景二 Oblique（606 帧训练集，yaw=15° 斜向安装，雷达高度 25m）

  Oblique 场景面临三重劣势：训练数据比 Frontal 少 8%（606 vs 660 帧）、斜向 15° 安装导致目标在图像和 BEV
  投影中产生系统性畸变、雷达安装高度 25m 使俯视角更大导致远端地面目标在雷达视场边缘。优化策略以激进补偿为主线：

  - BBox 回归 dist_weight
  分段：远端（>150m）3.0×、过渡段（100-150m）2.0×、中间段（50-100m）1.5×、近端（<50m）1.0×。设计特点是远端拉到 3.0×
  的极限权重——Oblique 的远端检测率和定位精度是最短板，必须用最强梯度信号驱动。中间段仅
  1.5×，因为数据量有限的情况下需要做取舍：将有限的模型容量和梯度预算向远端倾斜。
  - Heatmap Focal Loss 加权：全开。hw = heatmap_weights（值等于 dist_weight），远端目标热力图中心点获得 3.0×
  的分类损失权重。这与 Frontal 的关闭策略形成鲜明对比——原因在于 Oblique 数据量少（606
  帧），每帧的远端目标数也更少，不存在 Frontal 那样的"虹吸"条件（远端总正样本像素不足以压倒中间段）。相反，开启 heatmap
  加权恰恰利用了这一特性：在样本总量小的前提下，用高权重让每个远端目标产生更强的梯度信号，等效于对远端目标做了"上采样"，
  弥补其天然稀疏性。此外，斜向视角下目标在 BEV 热力图上更容易发生位置偏移，更高的 heatmap
  权重也帮助检测头更快收敛到正确的目标中心。
  - Gaussian Radius：与 Frontal 相同的 2.0×/1.5×/1.2×/1.0×，此处不求差异，因为 radius 扩大的收益（容错 depth
  噪声）在两个场景中是共通的物理问题。

  ==========================================一段内容总结==============================================================
  ● 总： 针对路侧场景远端漏检多、纵向定位精度差的问题，480 格配置从检测头和视图变换两个模块部署了四项优化。第一，BBox
  回归距离加权——在 BEVDepthHead 的 get_targets_single() 中为每个 GT 目标计算 dist_weight 标量并乘入 SmoothL1
  回归损失矩阵（bbox_weights = mask × code_weights × dist_weight），远端目标的回归损失被倍数放大，迫使优化器在参数更新时
  优先减小远端定位残差，等效于对稀疏的远端样本施加更大的学习率。第二，Gaussian Radius
  分段扩大——在生成训练热力图真值时，按目标纵向距离分四档将基础高斯半径放大
  1.0×~2.0×，远端目标的热力图正样本区域随之扩大，一方面增加 Focal Loss 中正样本占比以缓解极端正负不均衡，另一方面给予
  depth 噪声导致的 BEV 投影偏移更大的容错区间，避免远端目标因位置漂移被当作负样本惩罚。第三，Heatmap Loss
  距离加权——对热力图 Focal Loss 的逐像素权重矩阵施加与 BBox
  权重同源的距离系数，使远端目标的热力图中心像素获得更高的分类损失权重，强化远端分类监督信号密度。第四，Depth Loss
  距离感知——在视图变换模块的深度预测训练中，按 GT 深度 bin 给每像素 BCE 损失分配不同权重（远端像素 2.0×、中端 1.0×、近端
  0.5×），迫使 DepthNet 在雷达点云极度稀疏的远距离区域投入更多模型容量学习准确的深度分布，从源头改善 BEV
  特征中远端目标的空间编码质量，间接提升检测头对远端目标的定位能力。

  分： 两场景共享 Gaussian Radius（2.0×/1.5×/1.2×）和 Depth Loss 距离感知（2.0/1.0/0.5）的相同配置，但在 BBox 距离权重和
  Heatmap Loss 策略上形成对比。场景一 Frontal（660 帧，正向安装）采取保守稳健路线：BBox 权重分段为远端
  2.5×、过渡段和中间段同为 2.0×、近端 1.2×，中间段保持高权重以保护核心区域不因远端加码而退化；同时关闭 Heatmap
  距离加权（权重矩阵全 1），防止远端高斯核扩大后产生的大量正样本像素以高权重"虹吸" Focal Loss
  梯度，导致中间段目标分类精度下降，实现分类与回归的解耦优化。场景二 Oblique（606 帧，15°
  斜向安装）采取激进补偿路线：BBox 远端权重拉至 3.0×、中间段仅留存 1.5×，将有限的梯度预算向最短板倾斜；同时全开 Heatmap
  距离加权（值等于 dist_weight），因为在样本总量小的前提下不存在 Frontal
  那样的虹吸条件，反而能以高权重对稀疏的远端样本做等效"上采样"，且斜向视角下目标 BEV
  投影更易偏移，更强的逐像素分类监督有助于检测头快速收敛到正确位置。
